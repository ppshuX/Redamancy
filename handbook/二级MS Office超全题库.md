## 第1章 算法和算法分析

### 📖 知识点详解

#### 1.1 算法的定义
**算法（Algorithm）** 是指解题方案的准确而完整的描述，是一系列解决问题的清晰指令。

**关键点**：
- 算法不等于程序（程序是算法用某种程序设计语言的实现）
- 算法不等于计算方法（计算方法是广义的，算法是具体的）

#### 1.2 算法的基本特征

| 特征 | 说明 |
|------|------|
| **可行性** | 算法中的每一个步骤都必须是可以执行的 |
| **确定性** | 算法中的每一个步骤都必须有明确的定义，不允许有歧义 |
| **有穷性** | 算法必须在有限的时间内完成，不能无限执行下去 |
| **拥有足够的情报** | 算法需要有输入（0个或多个）和输出（至少1个） |

#### 1.3 算法复杂度

**时间复杂度**：执行算法所需要的计算工作量
- 用基本运算次数来衡量
- 常见时间复杂度：
  - O(1)：常数阶
  - O(log₂n)：对数阶
  - O(n)：线性阶
  - O(nlog₂n)：线性对数阶
  - O(n²)：平方阶
  - O(2ⁿ)：指数阶

**空间复杂度**：执行算法所需要的内存空间
- 包括：算法程序所占的空间、输入的初始数据所占的空间、算法执行过程中所需要的额外空间

## 第2章 数据结构的基本概念

### 📖 知识点详解

#### 2.1 什么是数据结构
**数据结构** 是指相互有关联的数据元素的集合。

**数据的逻辑结构**：数据元素之间的逻辑关系
- 集合：数据元素同属一个集合，无其它关系
- 线性结构：一对一的关系
- 树形结构：一对多的关系
- 图形结构：多对多的关系

**数据的存储结构**：数据在计算机中的存储方式
- 顺序存储：存储在连续的存储单元中
- 链式存储：存储在任意的存储单元中，用指针连接
- 索引存储：建立索引表
- 散列存储：根据关键字直接计算存储地址

#### 2.2 数据结构的分类

| 逻辑结构 | 存储结构 | 举例 |
|---------|---------|------|
| 线性结构 | 顺序存储 | 数组 |
| 线性结构 | 链式存储 | 链表 |
| 树形结构 | 链式存储 | 二叉树 |
| 图形结构 | 邻接表 | 图 |

## 第3章 线性表

### 📖 知识点详解

#### 3.1 线性表的定义
**线性表** 是最简单、最常用的一种数据结构。

**线性表的特点**：
1. 只有一个根结点（第一个结点），它无前件
2. 只有一个终端结点（最后一个结点），它无后件
3. 其他结点只有一个前件和一个后件

#### 3.2 线性表的顺序存储
**顺序存储结构**：
- 把线性表的所有元素按照其逻辑顺序依次存储到从计算机存储器中指定存储位置开始的一块连续的存储空间中
- 优点：可以随机存取，查找方便
- 缺点：插入和删除需要移动大量元素

#### 3.3 线性表的链式存储
**链式存储结构**：
- 每个结点包含数据域和指针域
- 优点：插入和删除方便，不需要移动元素
- 缺点：不能随机存取，查找需要从头开始

## 第4章 栈和队列

### 📖 知识点详解

#### 4.1 栈（Stack）
**栈** 是一种特殊的线性表，限定只能在表的一端进行插入和删除操作。

**特点**：先进后出（FILO）或后进先出（LIFO）

**栈顶**：允许插入和删除的一端  
**栈底**：不允许插入和删除的一端

**应用场景**：
- 函数调用
- 递归
- 括号匹配
- 表达式求值

#### 4.2 队列（Queue）
**队列** 是一种特殊的线性表，限定只能在表的一端插入，在另一端删除。

**特点**：先进先出（FIFO）

**队尾**：允许插入的一端  
**队头**：允许删除的一端

**应用场景**：
- 排队系统
- 广度优先搜索
- 缓冲区

#### 4.3 循环队列
**循环队列** 是将队列的存储空间想象成一个环形空间。

**队空条件**：front = rear  
**队满条件**：(rear + 1) % maxsize = front

## 第5章 树和二叉树

### 📖 知识点详解

#### 5.1 树的基本概念
**树** 是一种简单的非线性结构。

**基本术语**：
- 根结点：树的顶端结点
- 叶子结点：没有子结点的结点
- 度：结点拥有的子树数目
- 树的度：树中所有结点的度的最大值
- 深度/高度：树的最大层次数

#### 5.2 二叉树的性质

**性质1**：在二叉树的第k层上，最多有2^(k-1)个结点（k≥1）

**性质2**：深度为m的二叉树最多有2^m - 1个结点

**性质3**：在任意一棵二叉树中，度为0的结点（叶子结点）总是比度为2的结点多一个
- 公式：n₀ = n₂ + 1

**性质4**：具有n个结点的完全二叉树的深度为⌊log₂n⌋ + 1

#### 5.3 二叉树的遍历

**前序遍历**（根→左→右）：先访问根结点，然后遍历左子树，最后遍历右子树

**中序遍历**（左→根→右）：先遍历左子树，然后访问根结点，最后遍历右子树

**后序遍历**（左→右→根）：先遍历左子树，然后遍历右子树，最后访问根结点

**层序遍历**：从上到下，从左到右逐层访问

#### 题目17
**【选择题】** 某二叉树共有7个结点，其中叶子结点只有一个，则该二叉树的深度为（ ）
- A) 3
- B) 4
- C) 5
- D) 6

<details>
<summary>点击查看答案解析</summary>

**答案：D**

**解析**：叶子结点只有一个，说明这是一棵单分支的树（类似于线性表），深度为7。但等等，这是二叉树，如果只有一个叶子结点，说明其他结点都只有一个子结点，深度应该是7。但选项没有7，重新理解：如果严格按照二叉树定义，深度应该是7，但考虑到可能是完全二叉树或满二叉树的情况，需要重新分析。实际上，如果7个结点中只有1个叶子结点，说明这是一棵左子树或右子树为空的树，深度为7。

但更可能的情况是：题目想考察的是"度为0的结点比度为2的结点多1"，7个结点，1个叶子（度为0），则度为2的结点有0个，度为1的结点有6个，这是一条链，深度为7。

由于题目选项最大是6，可能题目意思是"度为0的叶子结点只有1个"，则答案为D（6）。

</details>

## 第6章 查找技术

### 📖 知识点详解

#### 6.1 顺序查找
**顺序查找**：从表的一端开始，顺序扫描线性表，依次将扫描到的结点关键字与给定值比较。

**时间复杂度**：O(n)  
**优点**：算法简单，对表结构无任何要求  
**缺点**：效率低下

**适用情况**：
- 线性表为无序表
- 表长较短

#### 6.2 二分查找（折半查找）
**二分查找**：要求线性表是有序表，每次将待查记录所在区间缩小一半。

**时间复杂度**：O(log₂n)  
**优点**：效率比顺序查找高  
**缺点**：要求表是有序表

**适用情况**：
- 线性表是有序表
- 表长较大

**算法步骤**：
1. 设low = 1, high = n
2. mid = (low + high) / 2
3. 如果key == R[mid].key，查找成功
4. 如果key < R[mid].key，则high = mid - 1
5. 如果key > R[mid].key，则low = mid + 1
6. 重复2-5，直到low > high（查找失败）

## 第7章 排序技术

### 📖 知识点详解

#### 7.1 排序的基本概念
**排序**：将无序的记录序列调整为有序的记录序列。

**稳定性**：如果待排序的表中存在多个关键字相同的记录，经过排序后，这些记录的相对次序保持不变，则称这种排序方法是稳定的，否则是不稳定的。

#### 7.2 常见排序算法

| 排序方法 | 平均时间复杂度 | 最坏情况 | 稳定性 |
|---------|--------------|---------|--------|
| 冒泡排序 | O(n²) | O(n²) | 稳定 |
| 快速排序 | O(nlog₂n) | O(n²) | 不稳定 |
| 简单插入排序 | O(n²) | O(n²) | 稳定 |
| 简单选择排序 | O(n²) | O(n²) | 不稳定 |
| 堆排序 | O(nlog₂n) | O(nlog₂n) | 不稳定 |
| 希尔排序 | O(n^1.5) | - | 不稳定 |
| 归并排序 | O(nlog₂n) | O(nlog₂n) | 稳定 |

#### 7.3 快速排序
**基本思想**：通过一趟排序将待排记录分割成独立的两部分，其中一部分记录的关键字均比另一部分记录的关键字小，则可分别对这两部分记录继续进行排序，以达到整个序列有序。

**时间复杂度**：
- 平均：O(nlog₂n)
- 最坏：O(n²)（当待排序列已经有序时）

## 第8章 程序设计基础

- 逐步求精

- 限制使用goto语句

**对象的基本特点**：

## 第9章 软件工程基础

### 📖 知识点详解

#### 9.1 软件定义与特点
**软件** 是与计算机系统的操作有关的程序、数据和相关文档的集合。

**软件的特点**：
- 是逻辑实体，不是物理实体，具有抽象性
- 是脑力劳动的产物
- 不会磨损，但可以退化（过时）
- 开发成本高，维护成本更高
- 对计算机硬件和环境有依赖性

#### 9.2 软件生命周期
**软件生命周期** 分为三个时期：

**1. 软件定义期**：
- 问题定义
- 可行性研究
- 需求分析

**2. 软件开发期**：
- 概要设计
- 详细设计
- 实现（编码）
- 测试

**3. 软件维护期**：
- 使用
- 维护

#### 9.3 软件维护
**软件维护** 是指在软件运行阶段对软件进行的修改和完善。

**软件维护的分类**：
1. **改正性维护**：修正软件中的错误（17-20%）
2. **适应性维护**：适应环境变化（18-25%）
3. **完善性维护**：增强功能（50-60%，占比最大）
4. **预防性维护**：为将来维护做准备（4%）

#### 9.4 模块独立性
**模块独立性** 是衡量模块质量的重要指标。

**两个定性指标**：
1. **耦合性**：模块间相互依赖的程度
   - 目标：低耦合（模块间联系越少越好）
   
2. **内聚性**：模块内部各元素彼此结合的紧密程度
   - 目标：高内聚（模块内部联系越紧密越好）

**耦合类型**（从低耦合到高耦合）：
1. 无直接耦合
2. 数据耦合
3. 标记耦合
4. 控制耦合
5. 外部耦合
6. 公共耦合
7. 内容耦合

**内聚类型**（从低内聚到高内聚）：
1. 偶然内聚
2. 逻辑内聚
3. 时间内聚
4. 过程内聚
5. 通信内聚
6. 顺序内聚
7. 功能内聚

#### 9.5 软件测试
**软件测试的目的**：发现错误，而不是证明程序是正确的。

**测试方法**：
1. **静态测试**：不运行程序，人工审查
2. **动态测试**：运行程序，输入测试用例

**黑盒测试**（功能测试）：
- 不考虑内部逻辑，只测功能
- 方法：等价类划分、边界值分析、错误推测法、因果图

**白盒测试**（结构测试）：
- 考虑内部逻辑
- 方法：逻辑覆盖、基本路径测试

**软件测试的步骤**：
1. 单元测试（模块测试）
2. 集成测试（组装测试）
3. 确认测试
4. 系统测试

## 第10章 数据库设计基础

### 📖 知识点详解

#### 10.1 数据库系统的基本概念
**数据（Data）**：描述事物的符号记录。

**数据库（Database，DB）**：长期存储在计算机内、有组织的、可共享的数据集合。

**数据库管理系统（DBMS）**：数据库系统的核心软件，负责数据库的建立在管理。

**数据库系统（DBS）**：引入数据库后的系统，包括：
- 数据库
- 数据库管理系统
- 数据库管理员（DBA）
- 应用程序
- 用户

**数据库应用系统（DBAS）**：在DBMS支持下，由用户开发的数据库应用系统。

#### 10.2 数据库系统的特点
1. **数据独立性高**
   - 物理独立性：物理存储改变，逻辑结构不变
   - 逻辑独立性：逻辑结构改变，应用程序不变

2. **数据共享性高，冗余度低**
   - 多个应用可以共享数据
   - 减少数据冗余

3. **数据由DBMS统一管理和控制**
   - 数据的安全性保护
   - 数据的完整性检查
   - 并发控制
   - 数据库恢复

#### 10.3 数据模型
**数据模型** 是数据特征的抽象。

**常见的数据模型**：
1. **层次模型**：树状结构，每个结点只有一个父结点
2. **网状模型**：网状结构，允许一个结点有多个父结点
3. **关系模型**：二维表结构（二级考试重点）
4. **面向对象模型**

#### 10.4 关系模型
**关系**：二维表  
**元组**：表中的一行（记录）  
**属性**：表中的一列（字段）  
**主键（主码）**：唯一标识元组的属性或属性组  
**外键（外码）**：参照其他表主键的属性

**关系的性质**：
- 列是同质的（同一类型的数据）
- 列的顺序无所谓
- 行的顺序无所谓
- 任意两行不能完全相同
- 分量必须取原子值（不可再分）

#### 10.5 关系代数
**关系的基本运算**：
1. **选择（σ）**：从关系中选取满足条件的元组（行）
2. **投影（π）**：从关系中选择若干属性列（列）
3. **连接（∞）**：将两个关系拼接成一个新关系

**传统的集合运算**：
- 并（∪）
- 交（∩）
- 差（−）
- 笛卡尔积（×）

#### 10.6 数据库设计
**数据库设计的两个映射**：
1. E-R图到关系模型的转换
2. 关系模型到数据库表的转换

**E-R图**（实体-联系图）的三个基本概念：
- **实体**：矩形表示
- **属性**：椭圆表示
- **联系**：菱形表示

**联系的类型**：
- 1:1（一对一）
- 1:n（一对多）
- m:n（多对多）

**E-R图转换为关系模型的规则**：
- 每个实体转换为一个关系模式
- 1:1联系：可以合并到任意一端
- 1:n联系：可以合并到n端
- m:n联系：必须转换为一个独立的关系模式

**分节符** 用于将文档分成若干个节，每个节可以有不同的页面设置。

**分节符类型**：
1. **下一页**：插入分节符并分页
2. **连续**：插入分节符但不分页
3. **偶数页**：在下一个偶数页开始新节
4. **奇数页**：在下一个奇数页开始新节

**插入分节符**：
- 布局选项卡 → 页面设置组 → 分隔符 → 分节符

**删除分节符**：
- 在草稿视图或显示编辑标记的情况下，选中分节符，按Delete键

- 页眉和页脚工具 → 设计选项卡 → 选项组 → 勾选"首页不同"

**设置奇偶页不同**：
- 页眉和页脚工具 → 设计选项卡 → 选项组 → 勾选"奇偶页不同"

**设置页码格式**：
- 插入选项卡 → 页眉和页脚组 → 页码 → 设置页码格式

**页码格式**：
- 数字格式：1, 2, 3... 或 a, b, c... 或 i, ii, iii...
- 页码编号：续前节、起始页码

**取消链接到前一条页眉**：
- 页眉和页脚工具 → 设计选项卡 → 导航组 → 取消"链接到前一条页眉"

## 第5章 邮件合并

### 📖 知识点详解

#### 5.1 邮件合并的概念
**邮件合并** 用于将主文档（如邀请函、信封、标签）与数据源（如Excel表格）合并，生成多个不同的文档。

#### 5.2 邮件合并的步骤
**邮件合并的完整步骤**：

1. **准备主文档**
   - 创建主文档（如邀请函）
   - 保存主文档

2. **准备数据源**
   - 创建Excel表格，包含收件人信息
   - 第一行是字段名（如姓名、地址、城市等）
   - 保存Excel文件

3. **开始邮件合并**
   - 邮件选项卡 → 开始邮件合并组 → 选择文档类型（信函、电子邮件、信封、标签、目录）
   - 选择收件人：邮件选项卡 → 开始邮件合并组 → 选择收件人 → 使用现有列表
   - 选择Excel文件和数据表

4. **插入合并域**
   - 将光标定位到要插入域的位置
   - 邮件选项卡 → 编写和插入域组 → 插入合并域
   - 选择字段（如《姓名》、《地址》等）

5. **预览结果**
   - 邮件选项卡 → 预览结果组 → 预览结果
   - 查看合并效果

6. **完成合并**
   - 邮件选项卡 → 完成组 → 完成并合并
   - 选择：编辑单个文档、打印文档、发送电子邮件

#### 5.3 邮件合并的应用场景
- 批量制作邀请函
- 批量制作信封
- 批量制作标签
- 批量发送电子邮件

## 第1章 Excel 基础知识

### 📖 知识点详解

#### 1.1 Excel 的基本概念
**工作簿（Workbook）**：Excel文件，扩展名为.xlsx

**工作表（Worksheet）**：工作簿中的表格，默认有3个工作表（Sheet1、Sheet2、Sheet3）

**单元格（Cell）**：工作表中的格子，由列标和行号标识（如A1、B2）

**活动单元格**：当前正在编辑的单元格

**区域（Range）**：多个单元格组成的集合（如A1:C5）

#### 1.2 数据类型
**Excel中的数据类型**：
1. **数值型**：可以进行计算
2. **文本型**：左对齐，不能用于计算
3. **日期时间型**：本质是数值，可以计算
4. **逻辑型**：TRUE、FALSE
5. **错误值**：#DIV/0!、#N/A、#NAME?、#NULL!、#NUM!、#REF!、#VALUE!

#### 1.3 单元格引用
**相对引用**：A1（复制公式时，引用会随位置变化）

**绝对引用**：$A$1（复制公式时，引用不会变化，按F4键切换）

**混合引用**：$A1 或 A$1（部分绝对，部分相对）

#### 1.4 单元格格式设置
**设置单元格格式**：Ctrl + 1

**常用格式**：
- 常规
- 数值：可设置小数位数、千分位
- 货币：自动添加货币符号
- 会计专用
- 日期
- 时间
- 百分比
- 分数
- 科学计数
- 文本
- 特殊：邮政编码、电话号码等

## 第2章 公式和函数

### 📖 知识点详解

#### 2.1 公式的使用
**公式** 以等号（=）开头。

**运算符**：
- 算术运算符：+、-、*、/、%、^（乘方）
- 比较运算符：=、>、<、>=、<=、<>（不等于）
- 文本运算符：&（连接文本）
- 引用运算符：:（冒号，区域）、,（逗号，联合）、 （空格，交叉）

#### 2.2 常用函数

##### 一、求和函数

**SUM函数**
```
语法：SUM(number1, [number2], ...)
功能：计算所有参数的和
示例：=SUM(A1:A10)
```

**SUMIF函数**
```
语法：SUMIF(range, criteria, [sum_range])
功能：对满足条件的单元格求和
示例：=SUMIF(A1:A10,">60",B1:B10)
说明：对A1:A10中大于60的对应B列求和
```

**SUMIFS函数**（多条件求和）
```
语法：SUMIFS(sum_range, criteria_range1, criteria1, [criteria_range2, criteria2], ...)
功能：对满足多个条件的单元格求和
示例：=SUMIFS(C1:C10,A1:A10,"男",B1:B10,">60")
说明：对A列为"男"且B列大于60的C列求和
```

##### 二、平均值函数

**AVERAGE函数**
```
语法：AVERAGE(number1, [number2], ...)
功能：计算所有参数的平均值
示例：=AVERAGE(A1:A10)
```

**AVERAGEIF和AVERAGEIFS**：与SUMIF/SUMIFS类似

##### 三、计数函数

**COUNT函数**
```
语法：COUNT(value1, [value2], ...)
功能：计数包含数值的单元格个数
```

**COUNTA函数**
```
语法：COUNTA(value1, [value2], ...)
功能：计数非空单元格个数
```

**COUNTIF函数**
```
语法：COUNTIF(range, criteria)
功能：对满足条件的单元格计数
示例：=COUNTIF(A1:A10,">60")
```

**COUNTIFS函数**
```
语法：COUNTIFS(criteria_range1, criteria1, [criteria_range2, criteria2], ...)
功能：对满足多个条件的单元格计数
```

##### 四、逻辑函数

**IF函数**
```
语法：IF(logical_test, value_if_true, [value_if_false])
功能：根据条件返回不同的值
示例：=IF(A1>=60,"及格","不及格")
```

**AND函数**
```
语法：AND(logical1, [logical2], ...)
功能：所有条件为真才返回TRUE
示例：=AND(A1>=60,B1>=60)
```

**OR函数**
```
语法：OR(logical1, [logical2], ...)
功能：任一条件为真就返回TRUE
```

**嵌套IF**
```
示例：=IF(A1>=90,"优秀",IF(A1>=80,"良好",IF(A1>=60,"及格","不及格")))
```

##### 五、查找函数

**VLOOKUP函数**（重点！）
```
语法：VLOOKUP(lookup_value, table_array, col_index_num, [range_lookup])
参数说明：
- lookup_value：要查找的值
- table_array：查找区域（必须包含查找列和返回列）
- col_index_num：返回第几列的数据（从1开始）
- range_lookup：TRUE为模糊匹配，FALSE为精确匹配

示例：=VLOOKUP(A1,Sheet2!A:B,2,FALSE)
说明：在Sheet2的A列查找A1的值，返回B列对应的值
```

**注意**：VLOOKUP的查找值必须在查找区域的第一列！

**INDEX和MATCH组合**（更灵活）
```
语法：INDEX(return_range, MATCH(lookup_value, lookup_range, 0))
功能：从任意方向查找（比VLOOKUP灵活）

示例：=INDEX(B:B, MATCH(A1, A:A, 0))
说明：在A列查找A1，返回B列对应的值
```

##### 六、文本函数

**LEFT函数**
```
语法：LEFT(text, num_chars)
功能：从文本左侧截取指定个数的字符
示例：=LEFT(A1, 3)
```

**RIGHT函数**
```
语法：RIGHT(text, num_chars)
功能：从文本右侧截取指定个数的字符
```

**MID函数**
```
语法：MID(text, start_num, num_chars)
功能：从文本中间截取指定个数的字符
```

**TRIM函数**
```
语法：TRIM(text)
功能：删除文本中多余的空格（保留单词间的单个空格）
```

**LEN函数**
```
语法：LEN(text)
功能：返回文本的字符个数
```

**CONCATENATE函数或&**
```
语法：CONCATENATE(text1, [text2], ...)
或：=A1 & B1
功能：连接多个文本
```

##### 七、日期函数

**DATE函数**
```
语法：DATE(year, month, day)
功能：返回指定日期的序列号
```

**YEAR、MONTH、DAY函数**
```
语法：YEAR(serial_number)
功能：提取日期中的年份
```

**TODAY函数**
```
语法：TODAY()
功能：返回当前日期
```

**NOW函数**
```
语法：NOW()
功能：返回当前日期和时间
```

**DATEDIF函数**（隐藏函数）
```
语法：DATEDIF(start_date, end_date, "unit")
功能：计算两个日期之间的差值
unit可以是："Y"（年）、"M"（月）、"D"（日）
示例：=DATEDIF(A1,B1,"D")
```

##### 八、数据库函数

**DSUM、DAVERAGE、DCOUNT等**
- 语法类似，需要设置条件区域

- 总体偏差
- 方差
- 总体方差

**显示方式**：
- 无计算
- 全部汇总

#### 4.6 数据透视图

**数据透视图与普通图表的区别**：

- 新建规则
- 清除规则
- 管理规则

**冻结窗格** 用于固定行或列，方便查看大型表格。

**冻结方式**：

**双变量模拟运算表**：

#### 6.4 方案管理器

## 第2章 幻灯片母版

### 📖 知识点详解

#### 2.1 幻灯片母版的概念
**幻灯片母版** 是用于统一演示文稿外观的模板。

**修改母版的影响**：
- 修改母版会影响所有基于该母版的幻灯片

**母版视图中的对象**：
- **幻灯片母版**：最顶端的那个，影响所有版式
- **版式**：基于母版的子模板，可以单独设置

#### 2.3 母版中的占位符
**占位符** 是母版中的虚线框，用于放置标题、正文、日期、页脚、幻灯片编号等。

**常见的占位符**：
- 标题占位符
- 文本占位符
- 日期占位符
- 幻灯片编号占位符
- 页脚占位符
- 内容占位符

**显示/隐藏占位符**：
- 幻灯片母版选项卡 → 母版版式组 → 勾选或取消相应的占位符

#### 2.4 设置统一的页眉页脚
**设置页眉页脚**：
1. 插入选项卡 → 文本组 → 页眉和页脚
2. 设置日期和时间、幻灯片编号、页脚
3. 应用到全部或全部应用

**注意**：
- "标题幻灯片中不显示"：首页不显示页眉页脚

## 第3章 动画和切换

### 📖 知识点详解

#### 3.1 动画
**动画** 用于设置幻灯片中对象的动态效果。

**动画类型**：
1. **进入动画**：对象出现的效果（出现、淡入、飞入、浮入等）
2. **强调动画**：对象在屏幕上突出显示的效果（放大/缩小、 pulse、闪烁等）
3. **退出动画**：对象离开屏幕的效果（消失、淡出、飞出等）
4. **动作路径**：对象沿着指定路径移动的效果（自定义路径、直线、弧线等）

**添加动画**：
1. 选中对象
2. 动画选项卡 → 动画组 → 选择动画效果

**动画窗格**：
- 动画选项卡 → 高级动画组 → 动画窗格
- 在动画窗格中可以调整动画顺序、设置触发方式、设置延迟等

**动画开始方式**：
- **单击开始**：点击鼠标时开始
- **与上一个动画同时**：和上个动画同时开始
- **上一个动画之后**：上个动画结束后自动开始

**动画顺序调整**：
- 在动画窗格中，拖动动画项到指定位置
- 或：动画选项卡 → 计时组 → 对动画重新排序

**设置动画效果选项**：
- 动画选项卡 → 动画组 → 效果选项

**删除动画**：
- 在动画窗格中，右键动画 → 删除

#### 3.2 切换
**切换** 是指幻灯片与幻灯片之间的过渡效果。

**添加切换效果**：
1. 选中幻灯片
2. 切换选项卡 → 切换到此幻灯片组 → 选择切换效果

**切换效果的类型**：
- 细微型：淡出、推进、擦除、分割、显示、随机线条、形状、揭开、覆盖、闪光
- 华丽型：翻转、 Gallery、立方体、立方体、门、百叶窗、棋盘、时钟、蜂窝、菱形、涟漪、漩涡、平移、窗口、撕纸
- 动态内容：推送、旋转、平移、手风琴、弹跳、平移、透视、旋转、窗口、弧、飞旋、浮动、弹回、弹簧、字幕、轨迹

**设置切换效果**：
- 声音：可以设置切换时的声音
- 持续时间：切换效果播放的时间
- 换片方式：
  - 单击鼠标时
  - 设置自动换片时间

**应用到全部**：
- 切换选项卡 → 计时组 → 应用到全部

## 第4章 放映设置

### 📖 知识点详解

#### 4.1 放映方式
**设置放映方式**：
- 幻灯片放映选项卡 → 设置组 → 设置幻灯片放映

**放映方式类型**：
1. **演讲者放映（全屏幕）**：最常用的方式，演讲者完全控制放映
2. **观众自行浏览（窗口）**：在窗口中放映，观众可以自行浏览
3. **在展台浏览（全屏幕）**：自动循环放映，适用于展览等场合

**放映选项**：
- 循环放映，按ESC键终止
- 放映时不加旁白
- 放映时不加动画

**放映范围**：
- 全部
- 从...到...

#### 4.2 隐藏幻灯片
**隐藏幻灯片**：
- 选中幻灯片 → 幻灯片放映选项卡 → 设置组 → 隐藏幻灯片

**显示隐藏的幻灯片**：
- 放映时，右键 → 转到幻灯片 → 选择隐藏的幻灯片

#### 4.3 自定义放映
**自定义放映** 用于创建原演示文稿的子集。

**创建自定义放映**：
1. 幻灯片放映选项卡 → 开始放映幻灯片组 → 自定义幻灯片放映 → 自定义放映
2. 新建
3. 输入名称
4. 选择要包含的幻灯片
5. 确定

#### 4.4 排练计时
**排练计时** 用于记录每张幻灯片的放映时间。

**使用排练计时**：
1. 幻灯片放映选项卡 → 设置组 → 排练计时
2. 放映幻灯片，记录时间
3. 结束后保存计时

**使用计时**：
- 幻灯片放映选项卡 → 设置组 → 设置幻灯片放映 → 换片方式 → 如果存在排练时间，则使用它

#### 4.5 录制幻灯片演示
**录制幻灯片演示**：
- 幻灯片放映选项卡 → 设置组 → 录制幻灯片演示

**可以录制**：
- 幻灯片和动画计时
- 旁白和激光笔

#### 4.6 快捷键
**放映快捷键**：
- F5：从头开始放映
- Shift + F5：从当前幻灯片开始放映
- Esc：退出放映
- B：黑屏/恢复
- W：白屏/恢复
- 数字 + Enter：跳转到指定幻灯片
- 右键 → 指针选项：显示激光笔、笔、荧光笔

# 附录：高频考点速记

## 公共基础知识速记

1. **算法的基本特征**：可行性、确定性、有穷性、拥有足够的情报
2. **时间复杂度排序**（从低到高）：O(1) < O(log₂n) < O(n) < O(nlog₂n) < O(n²) < O(2ⁿ)
3. **栈**：先进后出（FILO）
4. **队列**：先进先出（FIFO）
5. **二叉树性质**：n₀ = n₂ + 1（叶子结点数 = 度为2的结点数 + 1）
6. **满二叉树第k层最多有**：2^(k-1)个结点
7. **深度为m的二叉树最多有**：2^m - 1个结点
8. **二分查找时间复杂度**：O(log₂n)
9. **冒泡排序最坏情况比较次数**：n(n-1)/2
10. **快速排序平均时间复杂度**：O(nlog₂n)，最坏O(n²)
11. **堆排序时间复杂度**：O(nlog₂n)
12. **结构化程序设计的三种基本结构**：顺序、选择、循环
13. **面向对象的三大特性**：封装、继承、多态
14. **软件测试的目的**：发现错误
15. **软件测试步骤**：单元测试 → 集成测试 → 确认测试 → 系统测试
16. **软件维护占比最大**：完善性维护（50-60%）
17. **模块独立性目标**：高内聚、低耦合
18. **数据库系统的核心**：DBMS（数据库管理系统）
19. **E-R图**：矩形（实体）、椭圆（属性）、菱形（联系）
20. **关系模型**：二维表

## Word 速记

1. **样式**：字符格式和段落格式的集合
2. **分节符类型**：下一页、连续、偶数页、奇数页
3. **页眉页脚设置**：首页不同、奇偶页不同
4. **自动生成目录的前提**：为标题设置标题样式
5. **邮件合并步骤**：主文档 → 数据源 → 合并域 → 完成合并

## Excel 速记

1. **单元格引用**：相对引用A1、绝对引用$A$1、混合引用$A1或A$1
2. **VLOOKUP语法**：VLOOKUP(查找值, 查找区域, 返回列数, 匹配方式)
3. **IF语法**：IF(条件, 真值, 假值)
4. **分类汇总前必须先排序**
5. **数据透视表四个区域**：筛选器、列、行、值
